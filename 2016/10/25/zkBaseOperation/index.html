zk的基本操作

发表于 2016-10-25   |   分类于 zk   |  

在处理zk的问题时，需要对zk做些操作，来确定问题的缘由。而对zk的基本操作包括连接zk服务器、查看节点、删除节点、查看数据等。

连接zk server

需要安装zk，然后运行./zkCli.sh -server ip:port 即可连上zk 的服务器。接下来便可以在zk server上做些操作了

节点操作

  • 查看某个节点下面的子节点 ls /path
  • 查看某个节点的内容 get /path/node1
  • 删除某个节点 delete /path/node1
  • 给某个节点设置新值 set /path/node1 xxx

查看数据

查看zk 的数据可以通过事务日志或者snapshot查看。

  • 事务日志
java -cp /home/zookeeper/zookeeper-3.4.6.jar:/home/zookeeper/lib/slf4j-api-1.6.1.jar org.apache.zookeeper.server.LogFormatter log.700478c14 

日志内容：
10/12/16 8:52:28 PM CST session 0x357657c73714c38 cxid 0x4 zxid 0x70060241b create '/Duobao/task/quanziNotifyTask/cluster_lock0000000153,#31302e3132302e3234312e313632（节点内容）,v{s{31,s{'world,'anyone}}},T,154
  • snapshot
java -cp /home/zookeeper/zookeeper-3.4.6.jar:/home/zookeeper/lib/slf4j-api-1.6.1.jar org.apache.zookeeper.server.SnapshotFormatter snapshot.70049b8b5

snapshot文件内容：
    cZxid：创建节点时的ZXID
　　ctime：创建节点的时间
　　mZxid：节点最新一次更新发生时的zxid
　　mtime：最近一次节点更新的时间
　　pZxid：父节点的zxid
　　cversion：子节点更新次数
　　dataVersion：节点数据更新次数
　　aclVersion：节点acl更新次数
　　ephemeralOwner：如果节点为ephemeral节点则该值为sessionid，否则为0
　　dataLength：该节点数据的长度
